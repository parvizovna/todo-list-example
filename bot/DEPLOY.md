# Деплой бота на Railway (24/7)

## 1. Репозиторий на GitHub

Убедись, что проект запушен в GitHub (или другой Git-хостинг, который поддерживает Railway).

## 2. Создать проект в Railway

1. Зайди на [railway.app](https://railway.app) и войди (GitHub).
2. **New Project** → **Deploy from GitHub repo**.
3. Выбери репозиторий `todo-list-example` (или как он у тебя называется).
4. Railway создаст сервис из репозитория.

## 3. Настроить сервис под папку `bot`

У тебя в репозитории несколько частей (client, server, bot). Нужно сказать Railway, что деплоить только бота:

1. Открой созданный **сервис** (не проект).
2. Вкладка **Settings**.
3. В блоке **Source** найди **Root Directory**.
4. Укажи: **`bot`** (без слэшей).
5. Сохрани. Railway пересоберёт проект из папки `bot/`.

## 4. Переменные окружения

Без токена и ID бот не запустится. В том же сервисе:

1. Вкладка **Variables** (или **Settings** → Variables).
2. Добавь переменные:

| Переменная   | Значение |
|-------------|----------|
| `BOT_TOKEN` | Твой токен от @BotFather |
| `OWNER_ID`  | Твой Telegram user_id (число, например `1818964`) |

Сохрани. Railway сделает новый деплой с этими переменными.

## 5. Проверка

1. Вкладка **Deployments** — последний деплой должен быть в статусе **Success** (зелёный).
2. Открой **View Logs** — в логах не должно быть ошибок, бот должен писать что-то вроде запуска polling.
3. Напиши боту в Telegram `/start` — должен ответить.

## Важно

- **База SQLite** (`food_log.db`) на Railway **не сохраняется** между перезапусками (диск временный). Записи дневника будут теряться при редеплое. Если нужна постоянная база — потом можно добавить Railway Volume или внешнюю БД.
- Токен и `OWNER_ID` храни только в Variables в Railway, не коммить их в код.
- Бесплатный план Railway может засыпать сервис при неактивности; для стабильной работы 24/7 смотри актуальные условия на [railway.app/pricing](https://railway.app/pricing).

Готово: бот работает в облаке 24/7.
